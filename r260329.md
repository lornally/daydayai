You're looking for the `--dangerously-skip-permissions` flag. In VS Code 的 Claude Code 扩展中，可以这样配置：

打开 VS Code 设置（`Cmd+,`），搜索 `claude`，找到：

**Claude Code > Dangerous: Skip Permissions**

勾选即可。

或者在 `settings.json` 中添加：



```json
{
  "claudeCode.dangerous.skipPermissions": true
}
```
这会跳过所有工具调用的权限确认，包括文件写入、命令执行等。正如名字暗示的，用的时候注意风险 — 它会自动批准所有操作，不再弹窗确认。

另外还有一个: bypassPermissions, 

所以在 VS Code 扩展中，有两个相关设置：

1. **`claudeCode.allowDangerouslySkipPermissions`** — 允许跳过权限（前提开关）
2. **`claudeCode.initialPermissionMode`** — 新对话的初始权限模式，可选值：
   - `default` — 默认，每次都问
   - `acceptEdits` — 自动接受编辑
   - `plan` — 规划模式
   - `bypassPermissions` — 跳过所有权限确认

两个都设上就是完全自动批准。你现在的配置已经是最激进的了。
